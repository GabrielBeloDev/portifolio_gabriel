Existe uma estética que denuncia site feito no piloto automático: fonte Inter em tudo, cinza `slate` do Tailwind, azul de destaque, hero centralizado com texto em gradiente e três cards simétricos. Nenhum desses elementos é feio — o problema é que são o **caminho de menor esforço**, e por isso aparecem em todo lugar, iguais.

A defesa não é uma ferramenta: é trocar cada engrenagem por uma escolha que tenha motivo.

## Âmbar, não azul

| papel | dark | light |
| --- | --- | --- |
| fundo | `#0E1116` (tinta) | `#F6F5F2` (papel) |
| texto | `#E6E1D8` | `#1A1D21` |
| acento | `#FFB020` | `#A05C08` |

O acento âmbar veio do mundo do assunto: é cor de **anotação de engenharia**, de marca-texto em diagrama, de sinal de atenção em painel. Evita o azul-padrão e também os dois clichês vizinhos — o verde-terminal e o creme-com-terracota.

## IBM Plex, uma família só

Sans no corpo e na interface, Mono no código e nas etiquetas, Serif no texto longo dos posts. Uma família com pedigree técnico resolve os três papéis e tudo conversa — sem cair na fonte que todo gerador escolhe primeiro.

## A régua

Todo site precisa de um elemento que seja só dele. Aqui é a régua de numeração na lateral esquerda — os números de linha de um editor de código numerando as seções da página. É implementada só com contadores de CSS, zero JavaScript:

```css title="styles.css"
.ruled-page {
  counter-reset: section;
}

.ruled-num::before {
  content: counter(section, decimal-leading-zero);
  font-family: var(--font-mono);
}
```

Um detalhe que ninguém pediu: quando o cursor passa por uma seção, o número dela acende em âmbar. É o tipo de micro-decisão que nenhum template traz — e é exatamente aí que mora a diferença.
